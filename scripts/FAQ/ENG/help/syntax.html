General

  1. Comments are denoted similarly to many other languages: // and the pair / * * /
  2. Language ignores case.
  3. The symbol ";" means line splitting and end of statement. If there is nothing after the symbol, then this is an error.
  4. A preprocessor directive is denoted with a # at the beginning of a line. (at the time of this writing, only one #include directive is available)

Simple Data Types

Writing simple data types in expressions:

Integer - 1, 20, 4000;
Floating point - 3.14, .1
String - "stroka"
Constant - null, true, false, pi
Variable - my_var


Variables

Variable names must begin with a Latin letter or an underscore, and then can contain both numbers, letters and underscores. For example:

local myvar
local _myvar
local _my_var1

Below is a table of variable types.
Ad keyword Scope Value after function call
local Called function Not saved
static Called function Saved
global Everywhere Saved
public Everywhere Saved

Note: public differs from a global variable only in the ability to access it outside the script through a list. Variables of this type are also involved in instancing.

When declaring, it is necessary to take into account:
  • If a constant or function with the same name exists, it is considered an error.
  • If a variable is accessible only from a certain function (i.e. local or static), then the coincidence of names among the variables of this function is checked.
  • If it is required to declare a variable of a global type (global or public), then the coincidence of names among global ones is checked. BUT! If a local or static variable with this name is declared, then it will be used instead of the global one. In this case, the compiler will display a name conflict warning.

Constants

Constants can be declared like regular variables. They can also be declared by reference to another constant, for example:

const a = 1
const b = a


There are also superconstants - constants declared by default. These are null, true, false, pi, etc.

Functions

Functions are declared with the keyword def function_name ( parameters ). The default parameter values ??can be not only simple data types, but also expressions. All parameters are local variables of the declared function. When declaring a function, keep in mind that if a function, constant, local or global variable with the same name is declared, it is considered an error.
The default expression in the parameter is evaluated only if the value of the parameter is not passed when the function is called.

Arrays

Arrays are created using square brackets. For example:

array = [10, SE_INT] // Create an array of 10 values or

array = [10, SE_FLOAT] // Create an array of 10 values or

array = [10, SE_STRING] // Create an array of 10 values

Access to an array, as in many other PLs, is carried out using an index:

a = array [0] // a = 1
b = array [1] // b = 2

To change the value of an array:

array [0] = rand (3)

The size of the array can be found using the len operator:

length = len array // length = 3

Although strings are not represented as arrays, their length can also be found using the len operator. (example: len "stroka")

Index

To get a pointer to a variable, you need to put "*" in front of the identifier. Pointers are used to pass the variable itself (and not its value) to the function argument. Example:

local r, g, b
get_color (* r, * g, * b)

local color = [r, g, b]

def get_color(r,g,b)
r = 255
g = 0
b = 255
end
